In the deployed system, there will be a server running on the central machine and a server running on a cloud machine.  However, while developing I would like both of these servers run on the local machine but target two different schemas in the database.  If the NODE_ENV is development then the server should look for an environment variable named XEROSCOUT_REMOTE.  if this variable exists, the this instance should act as the remote instance and use the REMOTE_ values in the .env file to connect to the database.  Also, when in this development mode, no API keys should be necessary for either server.  When the central is started, both servers should be started when in development mode.  

In addition there should be sync agent running on the central local version of the server that syncs data via rest APIs with the remote server.
